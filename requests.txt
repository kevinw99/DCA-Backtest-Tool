Need to make one more change to doc and code:
When it is consecutive buy,  only price lower than the last buy price can trigger a new buy.
If price rises above the last buy price, then no buy can be triggered. Only one of these 2 condition can reset
consecutive buy count to 0.
A. there is a sell
B. price rises above the average cost of all lots held.
Once consecutive buy count is reset to 0, then next buy can be at any price as long as other conditions are met.

This is true regardelss whether  Consecutive Incremental Buy Grid is Enabled or disabled.

1. Current Implementation:
   - The current implementation uses global default parameters for all tickers.
2. We want to implement ticker-specific default parameters, so that each stock can have its own set of
   optimized default parameters based on its historical performance and volatility.
3. need to add a button on parameter page (individual backtest mode) to "Save as Default". When clicked, it saves the
   current parameters as the default for that ticker. If it does not exist, it creates a new entry backtestDefaults
   .json. We want this to be edited manually if needed.

 4. The current "Reset to Default" button should reset to the ticker-specific default if it exists, otherwise fall back to
  global default.

{
  "AAPL": {

  },

    "maxLots": 8,
    "gridIntervalPercent": 15,
    "profitRequirement": 15,
    ...
  },
  ...
}
6. you can keep the global defaults in the code as fallback values. and when ticker-specific defaults are not
fouund or not suitable, the global defaults will be used.





I think there are 3 general scenarios for long DCA:
1. The price goes up steadily, along with some dips that large enough to trigger buys, and spikes large enough to
trigger sell but the  overall trend is up. In this case, the DCA will work
well and be better than buy and hold, as it capture the buy on the dips, and sell on the peaks, along with the
overall uptrend. The overall P/L % is positive and is better than buy and hold, and the capital deployed is also
efficient.
2. the price goes down steadily, along with some spikes, but the overall trend is down. In this case, the DCA will
not work well, as it will keep buying on the dips, and never sell, as the price never goes up enough to trigger the
 sell because of profit requirement. The overall P/L % is negative and is better than buy and hold, but there more
 lots held, so the capital deloyed is more, so the P/L is much larger negative number, much larger loss than buy
 and hold, as it could be just a lot for buy and hold.

3. the price goes up quite quickly, without much dips, therefore there is no buy trigger, and we miss the chance to
 hold the stock for larger profit.  In this case, the DCA will not work well, we it may completely miss the run up
 since there is no buy trigger, and the overall P/L % is much smaller than buy and hold.
 How would you detect these scenarios? is it impossible to detect them in advance since all of these are
 afterthought summary of what happened in the past? or is there some indicators/metrics that can be used to predict
  these?



  Next Phase: Real-Time Adaptive Strategy

  The current implementation provides ex-post analysis (after the backtest completes). The next phase is to
  implement real-time adaptation during the backtest run, which is described in the design.md file.

  Key Features for Next Phase:

  1. Periodic Scenario Detection - Detect scenario every N days during backtest
  2. Dynamic Parameter Adjustment - Adjust grid spacing, profit requirements, position sizing based on
  detected scenario
  3. Regime Change Detection - Detect when market conditions shift
  4. Adaptive Strategy Metrics - Track how parameters changed over time

def generate_sequence(n, start=0.0, firstDelta=0.05, end=0.7, ith=None):
    """
    Generate a monotonically increasing sequence with:
    - a0 = start
    - a(n-1) = end
    - If firstDelta is provided, a1 = start + firstDelta
    - Absolute increments increase
    - Relative increments decrease

    Parameters:
    n : int
        Length of the sequence (must be >= 3)
    start : float
        First value (default 0.0)
    firstDelta : float or None
        First increment (a1 - a0). If None, use natural quadratic scaling.
    end : float
        Last value (default 0.7)
    ith : int or None
        If specified, return (value, delta) for that index.
        Otherwise, return the full sequence.
    """

- Symbol: NVDA

- Source: Database (backend/stocks.db)

1. Get stock ID by symbol

## Backend API Endpoints
- GET /api/stocks/:symbol - Get stock information
- GET /api/stocks/:id/prices - Get historical price data
- POST /api/data/fetch - Trigger data fetch from external APIs

}
