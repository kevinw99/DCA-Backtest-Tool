# Documentation Rules

⛔ **NO .md FILES IN ROOT DIRECTORY** (except README.md, CLAUDE.md, and DEPLOYMENT.md)

---

## Where to Put New Documentation

**Status/Progress Updates:**
→ Update `docs/status/COMPREHENSIVE_STATUS.md` in place
→ Create snapshot in `docs/status/snapshots/` for milestones

**User Guides:**
→ `docs/guides/YOUR_GUIDE_NAME.md`

**Testing Documentation:**
→ `docs/testing/YOUR_TEST_NAME.md`

**Analysis/Research:**
→ `docs/analysis/YOUR_ANALYSIS_NAME.md`

**Design Decisions:**
→ `docs/design/YOUR_DESIGN_NAME.md`

**Reference Material:**
→ `docs/reference/YOUR_REFERENCE_NAME.md`

**Spec Directories:**
→ `.kiro/specs/##_NAME/` (numbered sequentially)

**Temporary/Historical:**
→ `docs/archive/YYYY-MM-DD_DESCRIPTION.md`

---

## Essential Root Files Only

**Allowed in root:**
- `README.md` - Project overview
- `CLAUDE.md` - Claude instructions
- `DEPLOYMENT.md` - Deployment guide
- `.docs-rules.md` - Documentation organization (this file)
- `.file-rules.md` - File organization rules

---

## Spec Directory Naming

All spec directories in `.kiro/specs/` must be numbered:

- Format: `##_descriptive-name` (e.g., `01_algo-performance-calculation`)
- Use two-digit zero-padded numbers (01, 02, ... 10, 11, etc.)
- Separate number from name with underscore
- Use kebab-case for descriptive names
- Check existing specs to determine the next available number

---

## This Prevents

- Clutter in root directory
- Lost documentation
- Duplicated information
- Unclear organization

**See [docs/README.md](docs/README.md) for detailed documentation guide**
