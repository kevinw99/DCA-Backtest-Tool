# File Organization Rules

⛔ **MINIMAL FILES IN ROOT DIRECTORY**

Only essential project files allowed in root. Everything else goes in organized subdirectories.

---

## Allowed in Root

**Essential Files Only:**
- `README.md` - Project overview
- `CLAUDE.md` - Claude instructions
- `DEPLOYMENT.md` - Deployment guide
- `.docs-rules.md` - Documentation organization
- `.file-rules.md` - File organization (this file)
- `package.json` - NPM configuration
- `package-lock.json` - NPM lock file
- `render.yaml` - Render deployment config
- `LICENSE` or `LICENSE.md` - License file
- `requests.txt` - User's active request tracking

**Configuration Files (dotfiles):**
- `.gitignore`, `.npmignore`, `.dockerignore`
- `.eslintrc*`, `.prettierrc*`, `.editorconfig`, `.jscpd.json`
- `.env.example` (NOT .env with secrets!)

---

## Where to Put New Files

### Scripts (.sh, .bash, .zsh)
→ `scripts/your-script.sh`

**Examples:**
- Utility scripts → `scripts/`
- Test runners → `test/scripts/`
- Server management → `scripts/server/`

### Test Files (.json, .js test files)
→ `test/` or `test/fixtures/`

**Examples:**
- Test configurations → `test/fixtures/test_aapl.json`
- Test results → `test/results/testResults.txt`
- Test data → `test/data/`

### JavaScript Files (.js)
→ `backend/`, `frontend/`, or `scripts/`

**Source code:**
- Backend API → `backend/`
- Frontend app → `frontend/`
- Utility scripts → `scripts/`

### Documentation (.md, .txt)
→ `docs/` (see [.docs-rules.md](.docs-rules.md))

**Categories:**
- Status → `docs/status/`
- Guides → `docs/guides/`
- Testing → `docs/testing/`
- Analysis → `docs/analysis/`
- Reference → `docs/reference/`
- Archive → `docs/archive/YYYY-MM-DD_description.md`

### Configuration (.json, .yaml, .yml)
→ `config/` or within feature directory

**Examples:**
- App config → `config/app.json`
- Test config → `test/fixtures/`
- Baseline metrics → `test/baseline-metrics.json`

**Exceptions (allowed in root):**
- `package.json` - NPM config
- `render.yaml` - Render deployment
- `.prettierrc.json` - Prettier config
- `.jscpd.json` - Copy/paste detector config

### Log Files (.log)
→ `logs/` or `backend/logs/`

**Never commit logs!** Add to `.gitignore`

### Data Files (.csv, .json results)
→ `data/` or `test/results/`

**Examples:**
- Test results → `test/results/`
- Generated data → `data/generated/`
- Backtest results → `data/backtest-results/`

## Naming Conventions

### Scripts
- **Bash/Shell:** `kebab-case.sh` (e.g., `restart-server.sh`)
- **Python:** `snake_case.py` (e.g., `colorize_session.py`)
- **JavaScript:** `kebab-case.js` or `camelCase.js`

### Documentation
- **Markdown:** `UPPER_SNAKE_CASE.md` (e.g., `DEPLOYMENT.md`)
- **Archive:** `YYYY-MM-DD_description.md`

### Tests
- **Test configs:** `test_*.json` (e.g., `test_aapl.json`)
- **Test results:** `*Results.txt` or `*-results.json`

### Configuration
- **Descriptive names:** `app-config.json`, `baseline-metrics.json`

## Examples

### ✅ Good Organization

```
/
├── README.md
├── CLAUDE.md
├── DEPLOYMENT.md
├── package.json
├── render.yaml
├── backend/
│   ├── server.js
│   └── services/
├── frontend/
│   └── src/
├── scripts/
│   ├── restart-server.sh
│   └── colorize_session.py
├── test/
│   ├── fixtures/
│   │   ├── test_aapl.json
│   │   └── test_batch.json
│   └── results/
│       └── testResults.txt
├── docs/
│   ├── guides/
│   ├── testing/
│   └── reference/
└── data/
    └── backtest-results/
```
